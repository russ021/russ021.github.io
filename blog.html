Beginner-Friendly Linux Distros

Jan 25, 2025

Making the switch to Linux can be both exciting and overwhelming, especially if you're coming from Windows or macOS. With so many distributions (distros) to choose from, picking the right one is crucial to ensuring a smooth and frustration-free experience. Below we cover why to switch, what to look for, and some beginner-friendly distros.

Why Switch to Linux?

  • Free and open source — most distros are free.
  • Security and privacy — less vulnerable to common malware.
  • Performance and stability — runs well on old and new hardware.
  • Customization — tailor desktop environment and tools.
  • Community support — forums and guides widely available.

What to Look for in a Beginner-Friendly Distro

  • Easy graphical installer
  • User-friendly interface
  • Good hardware support
  • Large software repository
  • Active documentation and community

Recommended Distros

  1. Ubuntu — popular, large community, GNOME desktop.
  2. Linux Mint — Windows-like interface (Cinnamon), good out-of-the-box experience.
  3. Pop!_OS — System76 distro, great for gaming and modern hardware.
  4. elementary OS — macOS-like, minimal and privacy-focused.

Git Guide

Feb 19, 2025

Setup

$ git config --global user.name "Your Name"
$ git config --global user.email "your-email@example.com"

Basic Workflow

  1. Clone a remote repo
  2. Make local changes
  3. git add > git commit
  4. git pull then git push

Common Commands & Notes

  • Review changes: git diff or git diff --staged
  • Restore file: git restore <file> or git restore --staged <file>
  • View history: git log
  • Switch branches: git switch <branch>
  • Create and switch: git switch -c <newbranch>
  • Delete local branch (safe): git branch -d <branch>
  • Force-delete local branch: git branch -D <branch>
  • Delete remote branch: git push origin --delete <branch>
  • Prune remote-tracking branches: git fetch --prune or git fetch --prune --all

Merge Example

# on topic branch
$ git commit -m "finished"
$ git switch main
$ git merge topic
$ git branch -d topic

States of Files

Untracked, Unmodified, Modified, Staged — commands move files between these states (examples in the original guide).

Stash, Rebase & Submodules

  • Stash: git stash, git stash pop, git stash list
  • Rebase: git rebase --continue (do not rebase published history unless you know what you're doing)
  • Submodules clone: git clone --recurse-submodules <repo>

Rust Data Types

Apr 13, 2025

Scalar Types

  • Integers: i8/i16/i32/i64/i128, u8/u16/u32/u64/u128, isize/usize (arch)
  • Floats: f32 (single), f64 (double; default)
  • Boolean: bool
  • Character: char (Unicode scalar)

Compound Types

  • Tuple: fixed-size heterogeneous, e.g. let t: (i32, f64, char) = (42, 3.14, 'a');
  • Array: fixed-size homogeneous, e.g. let a: [i32; 5] = [1,2,3,4,5];
  • Vector: growable, e.g. let v: Vec = vec![1,2,3];
  • Strings: String (growable) and &str (slice)
  • HashMap: key-value store via HashMap<K, V>

Special Types

  • Option<T> — optional values
  • Result<T, E> — error handling

Ownership & References

&T (immutable reference), &mut T (mutable), Box<T> (heap allocation).

User-Defined Types

Structs and enums, e.g. struct Point { x: i32, y: i32 }, enum Color { Red, Green, Blue }.

Algorithms

"An algorithm is a finite set of unambiguous instructions that, given some set of initial conditions, can be performed in a prescribed sequence to achieve a certain goal and that has a recognizable set of end conditions."

Deciphering the TCP/IP Stack

Mar 26, 2026

Understanding how data travels across the internet can often feel like looking into a black box. However, the **TCP/IP (Transmission Control Protocol/Internet Protocol)** suite provides the fundamental map for how these connections happen. By breaking down the core components—from low-level protocols like PPP to high-level port management—we can demystify the mechanics of our digital lives.

The Foundation: What is a Protocol?

At its simplest, a network protocol is a set of rules that governs how data is exchanged between devices. Without these rules, a computer wouldn't know how to interpret the electrical or optical signals it receives from another machine.

Diving into the Data Link: Point-to-Point Protocol (IPCP/PPP)

One of the most essential protocols for establishing a direct connection between two nodes is the **Point-to-Point Protocol (PPP)**. It manages several critical phases of a connection: - **LCP (Link Control Protocol):** Responsible for establishing, configuring, and testing the data-link connection. It ensures both ends of the "pipe" are compatible before data flows. - **Authentication (PAP vs. CHAP):** Security is built into the connection process. While **PAP** (Password Authentication Protocol) sends credentials in plain text, **CHAP** (Challenge Handshake Authentication Protocol) is more secure, using a 3-way handshake to verify identity without ever sending the password over the link. - **CCP (Compression Control Protocol):** To make the most of available bandwidth, CCP manages the compression of data as it moves through the PPP link.

Navigating the Network: Gateways and Ports

Once a connection is established, data needs to find its way to the correct destination. This is where **gateways** and **ports** come into play. A gateway acts as a "traffic cop," directing data between different networks, while ports serve as communication endpoints for specific services (e.g., HTTP on port 80, HTTPS on port 443). Understanding how these components work together helps us grasp how our requests to websites and online services are routed across the globe.

Efficiency vs. Reliability: UDP vs. TCP in VPNs

A common question in networking is why many VPNs (Virtual Private Networks) prefer **UDP** (User Datagram Protocol) over **TCP**. While TCP is known for its reliability—it checks to make sure every single packet arrives in the correct order—this "checking" process adds overhead and latency. **UDP**, on the other hand, is "connectionless." It sends data as fast as possible without waiting for acknowledgments. In a VPN tunnel, using UDP often results in much higher speeds because it avoids the "TCP-over-TCP" meltdown, where two different protocols are both trying to manage error correction simultaneously.

Summary

From the initial handshake of an LCP packet to the high-speed routing of UDP traffic through a VPN, the TCP/IP suite is a marvel of layered engineering. Whether you are troubleshooting a gateway in Linux or configuring a secure server, understanding these building blocks is the first step toward mastering the network.
